写在node之前

断断续续学习NodeJS已经有一小段时间了，混迹于各大论坛，徘徊在诸多博客，多多少少看了一些基础视频，知乎，SegmentFault等提供了很多学习方法，学习路线。每每看看想到这些，总是热血沸腾，站在前辈大神的肩膀上，沿着各种学习之线，Node之路可谓一片光明。可是往往事与愿违，想象是这么的美好，现实总会让你体无完肤。如果没有真正的吸收这些知识，走过这些道路，大神只能是让你仰望的大神，美好的学习线路只能印上别人的脚踪，这一切都和你没有五分钱的关系。
ok，为了实现伟大的目标，可以更随心所欲的遨游在node海洋中，我就要放下浮躁的心，不能走马观花，只是“欣赏”别人的code，更要多写自己的code，这样才能让自己真正理解，而不是好像明白，真正写的时候就下不了手了。
当然，论坛还得混，博客还得逛，视频还得看，方法还得实践，但最重要的是形成自己的学习系统，一步一个脚印，扎稳扎牢，这样才能获得属于自己的node.

howToUseHexo

发表于 2015-06-28   |   分类于 Hexo使用   |  

hexo

npm install hexo -g #安装
npm update hexo -g #升级
hexo init #初始化

常用命令

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面只public目录
hexo server #开启本地预览服务器
hexo deploy #将.deploy目录部署到github
hexo help #查看帮助
hexo version #查看hexo的版本

复合命令

hexo deploy -g #生成加部署
hexo server -g #生成加预览

服务器命令

hexo server #hexo会监视文件变动并自动更新，无需重启服务器
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义IP

命令简写

hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo p == hexo publish
hexo d == hexo deploy

部署步骤

  1. hexo clean #清除缓存，网页正常情况下可以忽略此条命令
  2. hexo generate #生成静态网页
  3. hexo deploy #开始部署
